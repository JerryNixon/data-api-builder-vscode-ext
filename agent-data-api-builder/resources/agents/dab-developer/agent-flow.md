# DAB Agent Flow

## Purpose

Document the high-level decision flow used by the DAB developer agent.

## Primary branches

1. Starting point: existing database vs new local environment.
2. Config path: CLI-driven updates vs direct config edits.
3. Runtime path: REST, GraphQL, MCP (as requested).
4. Hosting path: local (Docker/Aspire) vs Azure deployment.
5. Support path: validation failures and runtime troubleshooting.

## Quality gates

- Validate before start.
- Confirm health before success claims.
- Keep guidance minimal and actionable.
